## 二、核心特性：断网可用 + 边缘智能

| 场景 | 功能 | 说明 |
|-----|------|------|
| **断网** | ✅ 实时显示 | 抬手看屏幕，心率/血氧/步数实时更新 |
| **断网** | ✅ 数据记录 | 本地FLASH存储运动与健康数据 |
| **断网** | ✅ APP查看 | BLE同步，APP可看历史记录 |
| **断网** | ✅ 边缘AI推理 | 跌倒检测、心律失常初筛、运动识别全在本地运行 |
| **断网** | ✅ 本地告警 | 跌倒/心率异常时振动马达+蜂鸣器本地提醒 |
| **联网** | ✅ 大模型分析 | Wi-Fi上传数据，DeepSeek生成每日健康报告 |
| **联网** | ✅ AI训练计划 | 上传运动数据，DeepSeek返回个性化训练计划 |
| **联网** | ✅ 设备响应指令 | 大模型下发指令→手环振动/蜂鸣/屏幕提醒 |
| **联网** | ✅ 数据同步 | APP历史数据与云端同步 |

## 三、通信设计

| 通道 | 用途 | 协议 | 说明 |
|-----|------|------|------|
| **屏幕** | 本地显示 | I2C OLED | 断网实时查看 |
| **BLE** | APP同步 | Bluetooth LE | 实时推送+历史同步 |
| **Wi-Fi** | 大模型交互 | HTTP REST API | 联网时获取分析+训练计划 |
| **FLASH** | 本地存储 | SPI | 掉电保存运动与健康数据 |
| **GPIO** | 执行器控制 | PWM/高低电平 | 振动马达+蜂鸣器触达用户 |

## 五、软件功能模块

### 5.1 设备端（ESP32-S3）

| 功能 | 说明 | 断网可用 | 核心算法 |
|------|------|---------|---------|
| 心率检测 | max30102 PPG信号 + 时域/频域算法 | ✅ | 峰值检测+滑动滤波 |
| 血氧检测 | max30102 红光/红外双通道比值法 | ✅ | R值→SpO2查表 |

| 步数检测 | mpu6050 加速度幅值+峰值检测 | ✅ | 动态阈值步数算法 |
| 卡路里估算 | 基于步数+运动类型+心率综合估算 | ✅ | MET值×体重×时间 |
| 运动类型识别 | mpu6050数据→TensorFlow Lite Micro推理 | ✅ | 本地CNN模型 |
| 跌倒检测 | mpu6050加速度突变+SVM轻量模型 | ✅ | 边缘AI推理（ESP-DL） |
| 心律失常初筛 | PPG信号异常模式匹配 | ✅ | 本地规则引擎+轻量分类 |
| 睡眠时长+质量 | 体动监测+HRV综合分析 | ✅ | 体动频率+HRV评分 |
| 屏幕显示 | OLED实时显示当前数据 | ✅ | LVGL驱动 |
| 本地存储 | FLASH保存运动/健康记录 | ✅ | SPI文件系统 |
| BLE同步 | 实时推送数据至APP | ✅ | GATT Service |
| 执行器响应 | 振动马达+蜂鸣器响应AI指令/本地告警 | ✅ | GPIO PWM控制 |
| 大模型交互 | Wi-Fi上传数据、接收指令并执行 | ❌（需联网） | HTTP + JSON |

### 5.2 边缘AI推理引擎（ESP32-S3本地运行）

| 模型 | 输入 | 输出 | 技术栈 | 断网可用 |
|------|------|------|--------|---------|
| **跌倒检测** | mpu6050 3轴加速度时序(2秒窗口) | 跌倒/未跌倒 | ESP-DL + 轻量SVM | ✅ |
| **心律失常初筛** | PPG信号RR间期序列(30秒窗口) | 正常/疑似房颤/早搏 | 规则引擎 + 决策树 | ✅ |
| **运动类型识别** | mpu6050 6轴数据(3秒窗口) | 静止/走路/跑步/骑行 | TensorFlow Lite Micro | ✅ |

#### 5.2.1 跌倒检测设计

```
mpu6050 加速度采集 (100Hz)
       │
       ▼
  滑动窗口 (2秒, 步长0.5秒)
       │
       ▼
  特征提取: 加速度幅值SVM、倾角变化、冲击持续时间
       │
       ▼
  ESP-DL推理 (轻量SVM/MLP模型)
       │
       ├── 无跌倒 ──► 继续监测
       │
       └── 检测跌倒 ──► 立即振动+蜂鸣器告警
                       └── 10秒内无手动取消 → BLE推送紧急信号至APP
```

#### 5.2.2 运动类型识别流程

```
mpu6050 6轴数据采集 (50Hz)
       │
       ▼
  3秒滑动窗口 (步长1秒)
       │
       ▼
  特征提取: 加速度均值/方差/峰度、角速度能量、FFT频域特征
       │
       ▼
  TensorFlow Lite Micro CNN推理
       │
       ├── 静止 (0)
       ├── 走路 (1) → MET=3.5, 记录步数+距离
       ├── 跑步 (2) → MET=8.0, 记录步数+距离
       └── 骑行 (3) → MET=6.0
       │
       ▼
  卡路里 = MET × 体重(kg) × 时间(h)
```

## 七、电源管理与低功耗设计

### 7.1 供电方案

```
  Type-C 5V ──► TP4056 充电管理 ──► 200mAh 锂电池
                     │                     │
                     │              ┌──────┴──────┐
                     │              │  3.3V LDO   │
                     │              │  (AMS1117)  │
                     │              └──────┬──────┘
                     │                     │
                     │              ┌──────┴──────────────────┐
                     │              │  ESP32-S3 + 传感器 + 执行器│
                     │              └─────────────────────────┘
                     │
               充电状态LED
```

### 7.2 功耗估算

| 模式 | 功耗 | 说明 |
|------|------|------|
| **正常工作模式** | ~80mA | Wi-Fi+BLE+传感器全开+屏幕亮 |
| **运动监测模式** | ~45mA | BLE+传感器+边缘AI推理, 屏幕间歇亮 |
| **日常佩戴模式** | ~15mA | BLE间歇广播+传感器定时采样(每5分钟) |
| **浅度休眠** | ~2mA | BLE保持连接, 传感器关闭, 仅保持RTC |
| **深度休眠** | ~50μA | 仅RTC唤醒, 定时唤醒采集数据 |
| **充电中** | 充电电流~150mA, 约1.5小时充满 |

### 7.3 低功耗策略

| 策略 | 实现方式 | 预期收益 |
|------|---------|---------|
| **传感器间歇采样** | max30102 仅在需测量时开启(默认关闭,每小时测1次心率) | 节省~20mA常态功耗 |
| **屏幕自动熄屏** | 抬手亮屏(mpu6050检测), 5秒无操作熄屏 | 节省~15mA屏幕功耗 |
| **Wi-Fi按需连接** | 仅在需要上传数据/同步报告时连接,其他时间保持关闭 | 节省~60mA (Wi-Fi是最大功耗源) |
| **BLE间歇广播** | 日常模式每5秒广播一次,运动模式持续广播 | 平衡功耗与实时性 |
| **Deep Sleep调度** | 夜间睡眠时段(0:00-6:00)进入深度休眠,RTC每小时唤醒采样 | 夜间仅消耗~0.3mAh |
| **动态CPU频率** | 计算任务完成立即降频, 空闲时80MHz, 推理时240MHz | 减少动态功耗 |
| **边缘AI推理优化** | 使用ESP32-S3向量指令加速, 模型INT8量化 | 推理时间缩短,功耗降低 |

### 7.4 续航目标

| 使用场景 | 续航 |
|---------|------|
| 纯待机(深度休眠) | ~15天 |
| 日常佩戴(间歇监测) | ~3天 |
| 连续运动监测(1小时/天) | ~2天 |

### 8.1 每日报告生成流程

```

       │
       ▼

  · 睡眠时长 / 入睡时间 / 醒来时间 / 体动数据
  · 运动达标情况 / 异常事件
       │
       ▼
  通过Wi-Fi发送至 DeepSeek API (JSON)
       │
       ▼
  DeepSeek 分析 + 生成报告:
  ┌───────────────────────────────────┐
  │         📊 每日健康报告            │

  │                                   │
  │  心率: 静息68 | 平均98 | 最高145  │
  │  心率区间: 热身35% 燃脂45% 有氧20%│
  │                                   │
  │  睡眠: 7h23m | 质量评分 82/100    │
  │  入睡 23:15  醒来 6:38           │
```

```
       │
       ▼
  生成个性化训练计划:
  ┌───────────────────────────────────────┐
  │         🏃 明日训练计划                │
  │                                       │
  │  体能状态: 良好 (恢复指数 85%)         │
  │                                       │
```

## 十一、传感器融合算法

### 11.1 运动心率补偿（mpu6050 + max30102融合）
以下仅供参考
**设计思路**：用mpu6050检测运动状态，根据运动强度动态调整心率滤波策略

| 运动状态 | 运动强度 | 处理策略 | 采样率 |
|---------|---------|---------|--------|
| 静止 | < 0.1 | 强滤波，α=0.9 | 50Hz |
| 走路 | 0.1~0.5 | 中等滤波，α=0.7 | 100Hz |
| 跑步 | > 0.5 | 运动伪影消除算法 | 200Hz |

**实现要点**：
- 运动强度 = 加速度幅值平方和的滑动平均值
- 滤波：低通滤波，系数α根据运动强度调整
- 运动伪影消除：自适应滤波，参考加速度信号

---

### 11.2 血压估算（PPG脉搏波特征）

**设计思路**：从PPG波形提取特征，结合心率、HRV、个人参数估算血压

| 特征参数 | 说明 |
|---------|------|
| 心率(HR) | 静态心率，每分钟心跳数 |
| 心率变异性(HRV) | SDNN值，反映自主神经调节 |
| 年龄/体重 | 个人生理参数，用于校准 |

**估算公式**（仅供参考，需实际数据校准）：
```
收缩压 = 基础值 + 0.5×心率 + 0.2×HRV - 0.1×年龄 + 0.1×体重
舒张压 = 基础值 + 0.3×心率 + 0.1×HRV - 0.05×年龄
```

**实现要点**：
- 需要用户初次使用时进行个人校准
- 只提供相对趋势，不作为医疗诊断依据
- 连续监测3分钟以上取平均值

---

### 11.3 睡眠质量分析（体动 + 心率融合）

**设计思路**：结合体动数据和心率变异性综合评估睡眠质量

| 传感器 | 采样率 | 用途 |
|-------|-------|------|
| mpu6050 | 1Hz | 体动检测，判断睡眠深度 |
| max30102 | 每5分钟 | 心率/HRV，识别睡眠阶段 |

**融合逻辑**：
1. **入睡判断**：连续10分钟体动少 + 心率下降
2. **睡眠阶段**：
   - 体动少 + HRV高 → 深睡
   - 体动多 + HRV低 → 浅睡
3. **质量评分**（0-100）：
   - 睡眠时长权重40%
   - 深睡比例权重30%
   - 体动次数权重20%
   - HRV稳定性权重10%

**输出指标**：
- 睡眠时长（小时）
- 睡眠质量评分（0-100）
- 入睡/醒来时间
- 睡眠阶段分布
