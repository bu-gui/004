# 智能运动健康手环 - 软件部分实施计划

## 项目概述

软件部分包含三大子系统：
- **设备端（ESP32-S3）**：传感器驱动、边缘AI推理、通信、显示、执行器控制
- **APP端（Flutter）**：数据展示、历史记录、AI报告、BLE通信
- **云端（DeepSeek API）**：每日报告、训练计划、健康建议

---

## 一、技术栈选型

### 1.1 设备端 (ESP32-S3)

| 层面 | 技术选型 | 说明 |
|-----|---------|------|
| **开发框架** | ESP-IDF v5.x （推荐） / Arduino-ESP32 | ESP-IDF官方支持，性能更好；Arduino开发更快 |
| **传感器驱动** | 自主编写 I2C 驱动 | MAX30102 + MPU6050 寄存器操作 |
| **边缘AI** | ESP-DL (ESP-DL) + TensorFlow Lite Micro | 跌倒检测用ESP-DL；运动识别用TFLite Micro |
| **显示** | LVGL v8.x + SSD1306 驱动 | 轻量级GUI框架 |
| **文件系统** | SPIFFS / LittleFS | 存储配置 + 历史数据 |
| **BLE** | NimBLE (Bluetooth LE) | ESP-IDF原生BLE栈，低功耗 |
| **Wi-Fi** | ESP-IDF Wi-Fi API | HTTP Client连接DeepSeek API |
| **JSON解析** | cJSON / ArduinoJson | 数据序列化/反序列化 |
| **RTOS** | FreeRTOS (ESP-IDF内置) | 多任务调度 |

### 1.2 APP端 (Flutter)

| 层面 | 技术选型 | 说明 |
|-----|---------|------|
| **开发框架** | Flutter 3.x + Dart | 跨平台（iOS + Android） |
| **BLE通信** | flutter_blue_plus | Flutter BLE库 |
| **状态管理** | Provider / Riverpod | 推荐Riverpod |
| **图表展示** | fl_chart | 心率/步数趋势图 |
| **本地存储** | sqflite / Hive | 缓存历史数据 |
| **HTTP请求** | dio | 调用DeepSeek API |
| **UI组件** | Material Design 3 | 现代化UI风格 |

### 1.3 云端

| 层面 | 技术选型 | 说明 |
|-----|---------|------|
| **大模型** | DeepSeek API | 每日报告 + 训练计划 + 健康建议 |
| **数据格式** | JSON | 结构化数据交换 |
| **通信协议** | HTTPS REST API | 安全可靠 |

## 十二、关键调试技巧

### 设备端常见问题

```text
【I2C设备扫描不到】
  ├─► 检查上拉电阻（4.7kΩ是否接好？）
  ├─► 检查地址是否正确（MAX30102: 0x57, MPU6050: 0x68, SSD1306: 0x3C）
  └─► SDA/SCL引脚是否接反？示波器看波形

【心率数据不稳定】
  ├─► 增大MAX30102 LED电流（寄存器0x0C）
  ├─► 检查佩戴松紧度
  ├─► 降低采样率（建议50Hz~100Hz）
  └─► 添加软件滤波（滑动平均+带通滤波）

【BLE连接不稳定】
  ├─► 检查MTU协商（建议≥247 bytes）
  ├─► 减少Notification数据长度
  ├─► 缩小BLE广播间隔（100ms→40ms）
  └─► 避免Wi-Fi和BLE同时全速运行（天线切换问题）

【边缘AI推理失败】
  ├─► 确认模型已正确转换为INT8量化
  ├─► 确认输入tensor形状与模型一致
  ├─► 分配足够的PSRAM（模型占用）
  └─► 使用ESP-DL的示例代码对比测试
```
