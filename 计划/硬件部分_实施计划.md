# 智能运动健康手环 - 硬件部分实施计划

## 项目概述

基于 ESP32-S3 主控的智能运动健康手环，集成多传感器采集、边缘AI推理、BLE/Wi-Fi通信、OLED显示及执行器反馈。

## 三、Phase 1：原型验证（第1~2周）

### 第1周：电路原理图设计与元件验证

#### 3.1.1 任务分解

| 任务 | 具体内容 | 预计工时 | 产出物 |
|-----|---------|:--------:|-------|
| T1.1 主控最小系统 | ESP32-S3 电源/晶振/复位/下载电路 | 4h | 原理图模块 |
| T1.2 传感器接口 | MAX30102 I2C + MPU6050 I2C 连接 | 3h | 原理图模块 |
| T1.3 OLED接口 | SSD1306 I2C 连接 + 电平匹配 | 1h | 原理图模块 |
| T1.4 Flash接口 | W25Q32 SPI 连接 (CS/SCK/MOSI/MISO) | 1h | 原理图模块 |
| T1.5 执行器驱动 | 振动马达(PWM) + 蜂鸣器(PWM) 驱动电路 | 2h | 原理图模块 |
| T1.6 电源管理 | TP4056充电 + LDO稳压 + 电池保护 | 3h | 原理图模块 |
| T1.7 整体原理图集成 | 各模块拼接 + 去耦电容 + 按键 + 调试接口 | 2h | 完整原理图 |
| **合计** | | **16h** | |

#### 3.1.2 关键设计要点

```text
【电源树】
Type-C 5V ──► TP4056 ──► 锂电池 3.7V
                              │
                         AMS1117-3.3
                              │
                         ┌────┴──────────────┐
                         │ 3.3V 供电域         │
                         │ ESP32-S3 + 传感器   │
                         │ + OLED + Flash      │
                         └─────────────────────┘
【I2C总线】
ESP32-S3 GPIO18(SDA) ──► MAX30102 SDA
                        ──► MPU6050 SDA
                        ──► SSD1306 SDA
ESP32-S3 GPIO19(SCL) ──► MAX30102 SCL
                        ──► MPU6050 SCL
                        ──► SSD1306 SCL
（注意：I2C需4.7kΩ上拉电阻）

【SPI总线 (Flash)】
ESP32-S3 GPIO10(CS)  ──► W25Q32 CS
ESP32-S3 GPIO11(MOSI)──► W25Q32 MOSI
ESP32-S3 GPIO12(MISO)──► W25Q32 MISO
ESP32-S3 GPIO13(SCK) ──► W25Q32 SCK

【执行器PWM】
振动马达 ──► GPIO4 (LEDC PWM, 频率1kHz)
蜂鸣器   ──► GPIO5 (LEDC PWM, 频率2kHz)
```

#### 4.1.2 任务分解

| 任务 | 具体内容 | 预计工时 | 产出物 |
|-----|---------|:--------:|-------|
| T3.1 元器件封装确认 | 确认所有元件封装（SMD/插件） | 2h | 封装库准备 |
| T3.2 原理图转PCB | 网络表导入 + 元件布局规划 | 3h | PCB布局初版 |
| T3.3 PCB布局 | 按功能分区布局（电源区/传感器区/主控区） | 4h | 布局完成 |
| T3.4 PCB布线 | 电源线加粗 + 信号线走通 + 地平面 | 6h | 布线完成 |
| T3.5 DRC检查 + 覆铜 | 设计规则检查 + 顶层底层覆铜 | 1h | PCB文件完成 |
| **合计** | | **16h** | |

#### 4.1.3 PCB设计关键要点

```text
【布局原则】
┌────────────────────────────────────────────┐
│  [Type-C] [TP4056] [LDO]   ← 电源区（左上）  │
│                                            │
│  [ESP32-S3 核心模组]        ← 主控区（中央）  │
│                                            │
│  [MAX30102] [MPU6050]      ← 传感器区（右下）│
│                                            │
│  [OLED接口] [Flash]         ← 外设区（右中）  │
│                                            │
│  [马达] [蜂鸣器] [按键]      ← 执行器区（左下）│
└────────────────────────────────────────────┘

【布线要点】
1. 电源线宽 ≥ 0.8mm（承载200mA以上电流）
2. I2C信号线长度尽量等长，远离PWM信号
3. SPI信号线加地线屏蔽
4. 晶振尽可能靠近ESP32-S3
5. 模拟地（传感器）和数字地（主控）单点连接
6. MAX30102附近预留10μF+100nF去耦电容
7. 电池接口加ESD保护
8. 板子尺寸控制在 45mm × 25mm 以内（手环形制）
```

### 第4周：PCB打样与焊接

#### 4.2.1 任务分解

| 任务 | 具体内容 | 预计工时 | 产出物 |

#### 4.2.2 焊接顺序与验证

```text
第1步：焊接电源部分（TP4056 + LDO + 电容）
  └─► 验证：输入5V → 输出3.3V正常

第2步：焊接ESP32-S3模组 + 晶振 + 去耦电容
  └─► 验证：串口打印正常，可烧录程序

第3步：焊接传感器（MAX30102 + MPU6050）
  └─► 验证：I2C扫描到各设备地址

第4步：焊接OLED接口 + Flash
  └─► 验证：OLED显示 + Flash读写

第5步：焊接执行器 + 按键 + 电池接口
  └─► 验证：马达振动、蜂鸣器响、按键响应
```

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|-----|:---:|:----:|---------|
| ESP32-S3引脚不够 | 低 | 高 | 提前规划引脚分配，使用I2C复用减少引脚占用 |
| MAX30102 PPG信号质量差 | 中 | 中 | 做好模拟电源隔离；佩戴时增加遮光；软件滤波 |
| PCB打样周期延误 | 中 | 中 | 提前下单，预留缓冲时间；同时保留面包板作为后备 |
| 电池续航不足 | 中 | 高 | 尽早测量功耗，预留软件降频/Deep Sleep优化空间 |
| 焊接失误烧坏元件 | 低 | 中 | 多采购1~2套备用元件，焊接前万用表确认 |
| I2C总线冲突/干扰 | 低 | 中 | PCB布线I2C远离PWM；加4.7kΩ上拉；降低I2C频率至100kHz |
